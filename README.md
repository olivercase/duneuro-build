# duneuro-build

Reproducible build recipe for the local **DUNEuro** stack used by the iNOB
vagus-nerve forward model (see the [`iNOB`](https://github.com/olivercase/iNOB)
repo). This repo contains **only the build recipe** — scripts, patches, and
build logs. It does **not** contain the compiled `duneuropy` extension, the
`duneuro-src` checkout, or the Python virtualenv (those are large, machine-
specific, and regenerated by the scripts below).

## What gets built

DUNE 2.10 + duneuro + duneuro-py against Python 3.11, producing a `duneuropy`
extension module installed into a venv. duneuro is pinned to commit `8f344b4`
with `scripts/patches/duneuro-eigen5-dune210.patch` applied (Eigen-5
`jacobiSvd` + DUNE-2.10 all-codim `evaluate` fixes).

## Layout

| Path | Purpose |
|------|---------|
| `scripts/build_duneuro_local.sh` | Local macOS / Apple Silicon build (Homebrew, Apple clang, C++20, python3.11). Resumable; reuses existing clones/venv. |
| `cluster/build_duneuro.sh` | HPC / module-system build variant. |
| `scripts/patches/duneuro-eigen5-dune210.patch` | Source patch required to compile against Eigen 5 / DUNE 2.10. |
| `scripts/patches/README.md` | Notes on the patch. |
| `logs/` | Captured build logs from the reference build (May 2026), for diagnosing build failures. |

## Local build (macOS / Apple Silicon)

```bash
# Output lands in $DUNEURO_BASE (default /Volumes/UCL/duneuro_build):
#   $BASE/duneuro-src   source checkouts
#   $BASE/venv          python3.11 venv with duneuropy installed
bash scripts/build_duneuro_local.sh
```

Prerequisites (Homebrew): `eigen gmp metis superlu cmake python@3.11`.

## Running the forward solve

The build installs `duneuropy` into `$BASE/venv` (default
`/Volumes/UCL/duneuro_build/venv`). The forward solve runs from **that** venv,
so the [`iNOB`](https://github.com/olivercase/iNOB) package must be installed
into it as well — install it editable so the venv tracks your working copy:

```bash
BASE=/Volumes/UCL/duneuro_build
cd /path/to/iNOB
"$BASE/venv/bin/pip" install --no-deps -e .        # inob + entry points into the venv

# sanity check: duneuropy imports and the pipeline can see it
"$BASE/venv/bin/python" -c "import duneuropy, inob; print('duneuro + inob OK')"

# run the real solve with that interpreter's console scripts
"$BASE/venv/bin/inob-forward"                       # MEG leadfield
"$BASE/venv/bin/inob-calibrate"                     # self-contained sphere solve (quick check)
```

> The editable install records an **absolute** path to the iNOB checkout. If you
> later rename or move the iNOB repo, re-run `pip install --no-deps -e .` from
> the new location — otherwise `import inob` breaks while `duneuropy` still works.
